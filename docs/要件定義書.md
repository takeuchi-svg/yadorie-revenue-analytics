# 売上分析BI — 要件定義書（現行版）

**プロジェクト**: yadorie-revenue-analytics
**構成**: Supabase + GitHub + Vercel
**先行施設**: FRY（山の手ホテル）→ 後続で全27施設へ展開
**最終更新**: 本番デプロイ済み・実データ投入フェーズ

> 旧「本番環境構築_要件定義書」「要件定義書_Phase2」をベースに、現在の実装状況を反映して更新したもの。

---

## 1. 目的

旅館・ホテルの売上を多次元（チャネル/客室/プラン/曜日/喫食/居住地/GS/ADR帯/キャンセル/F&B/OTA）で
分析するBIダッシュボード。PMS（Staysee）・サイトコントローラ（Lincoln）・レート表のCSV/Excelを取り込み、
Supabaseに蓄積、mart層ビューで動的集計して表示する。

利用者: 当面は社内（竹内）のみ。認証は最小構成（email/password 1ユーザー）。

---

## 2. 技術スタック（実装済み）

| 項目 | 採用 |
|------|------|
| フレームワーク | Next.js 16 (App Router) + TypeScript |
| UI | Tailwind CSS（ダークテーマ固定。`/docs/UI仕様書.md`準拠）|

| ホスティング | Vercel（個人アカウント takeuchi-svg、mainブランチ自動デプロイ）|

## 6. 既知の課題・設計上の決定

1. **稼働率は予約データから算出しない**。Staysee予約情報に室数列が無く、部屋名にダイニング/宴会場が
   混在するため、`raw_room_sales`（販売数集計表）を正データとする。詳細は設計書§4.2。
2. 既存 mart_monthly_kpi / mart_daily の occ は予約nightsベースで不正確。OCC表示は mart_occupancy_* へ接続する。
3. localStorageは使用しない（CLAUDE.md方針）。施設選択はCookie。
