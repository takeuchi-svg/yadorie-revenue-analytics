> ⚠️ 更新（as-built）: ページ構成が古い。**現行の正は [`00_ドキュメント地図.md`](00_ドキュメント地図.md) のページ一覧**（実12ページ: 概要/売上分析/日別売上/レート/CXL/料飲/OTA/予実(yojitsu)/生産性/シフト/アップロード/設定）。本書の「9ページ」「On-hand」（=現 日別売上/daily）、§3.8 Tab2「Sheets CSV読込」（localStorage時代の遺物・廃止）、Upload対応ファイルに販売数集計表が無い等は古い記述。配色・フォーマッタ・チャート方針は引き続き有効。

**目的**: 本番環境のUIを、デモ環境のUI/UXに統一するための仕様書
**対象**: 本番環境を実装しているClaude Code
**スコープ**: UI/UXのみ（データ層・アーキテクチャ・localStorage実装には言及しない）

---

## 0. この仕様書の使い方

本書は **画面の見た目と操作性（UI/UX）だけ** を定義します。
データの取得方法・保存方法（Supabase接続等）は本書の対象外で、
別途の本番環境設計に従ってください。

本書のゴールは「**どのページに、どんなカードやチャートが、どんな配置・配色で並ぶか**」を
本番環境でデモと同一にすることです。

- 金額（大）: `¥15.91M`（百万単位、小数2桁）

## 3. ページ別UI仕様

### 3.1 Overview

月次実績のサマリページ。主要KPIとトレンドを一覧で把握。

**KPIカード（6枚、横並び）**

| KPI | 表示例 |
|-----|--------|
| 売上 | ¥15.91M |
| OCC（稼働率） | 73.2% |
| REVPAR | ¥60,972 |
| 客単価 | ¥35,052 |
| 予算達成率 | 111.4% |
| 同伴係数 | 2.38 |

各カードにYoY比較（▲▼ + %、色分け）を表示。

**チャート（2つ）**

| チャート | 種別 | 内容 |
|---------|------|------|
| monthlyTrendChart | bar + line | 当年売上（棒）vs 前年（線）、6ヶ月 |
| channelPieChart | doughnut | チャネル別売上構成比 |

**AI実績サマリカード**: グラデーション背景で中央に配置。表示のみ（接続なし）。

---

### 3.3 On-hand

オンハンド（今後の予約状況）を月別・日別で表示。

- 月別サマリテーブル（売上/室数/在庫/OCC/ADR/前年比）
- OCC比較チャート（当年 vs 前年、lineチャート）
- 売上比較チャート（bar + line）
- 日別カレンダーヒートマップ（OCCによる3段階色分け）

確定実績は通常色、オンハンド（将来）は `--yellow` 系で区別。

---

### 3.4 Rate Control

レートランク×残室数のヒートマップグリッド。

- レートランクグリッド（CSS Grid、3段階色分け：20-25 / 26-33 / 34+）
- コントロールフラグタイムライン（5フラグ：直前割/素泊まり/朝食付/2名カット/カード縛り）
- AIコメンタリー

ランク色分け：20-25=緑系、26-33=amber系、34+=赤系。残室2以下は赤字強調。

---

### 3.5 Cancel Analysis

キャンセル分析ページ。

- 4つのKPIカード（CXL率 / 件数 / 金額 / ネット予約）
- チャネル別積み上げ横棒（緑=確定、赤=取消）
- リードタイム分布チャート（10バケット）

### 3.6 F&B / Upsell

### 3.8 Upload

2タブ構成のデータ取込ページ。

**Tab 1: CSV取込（月次更新）** — 4ステップのウィザード形式

| ステップ | 内容 |
|---------|------|
| Step 1 | ドラッグ&ドロップエリア（7ファイル対応） |
| Step 2 | ファイル自動判定結果表示 + 手動修正可 |
| Step 3 | プログレスバー付き変換処理 |
| Step 4 | KPIプレビュー + ダッシュボード反映ボタン |

**対応ファイル（7種）**

| ファイル | 判定キーワード | 形式 |
|---------|--------------|------|
| Staysee 予約情報 | 「予約情報」 | .csv (cp932) |
| Staysee 基本商品 | 「基本商品情報」 | .csv |
| Staysee その他商品 | 「その他商品情報」 | .csv |
| Staysee 入金 | 「入金情報」 | .csv |
| Lincoln CSV（2本） | 「予約検索」 | .csv |
| レート表 | 「レートチェック」 | .xlsx |

Lincoln CSVはCI日ベース（日付範囲≤35日）と受信日ベースを自動判定。

**Tab 2: Sheets CSV読込**（※本番でSupabase運用する場合は不要な可能性あり。UI構造のみ参考）

## 6. 本番実装時の注意

- 本書は**見た目（UI/UX）の正**である。データの出所（Supabase等）は本書の範囲外。
- 既存の本番UIが本書と異なる場合、**本書に合わせて修正**する。
- ページ構成（9ページ・3グループ）、KPIカードの枚数と内容、チャートの種別と配置、
  配色（ダークテーマ + チャネル色）を、デモと完全に一致させることがゴール。
- データ層の実装（localStorage→Supabase等）は本書では変更しない。別途指示に従う。
