# YADORIE シフト・労務管理システム 仕様書（v1）

本書は実装（Claude Code）向けの技術仕様。要件は `要件定義書_シフト労務管理v1.md`、DDLは `ddl_shift_labor_v1.sql` を正とする。

---

## 1. 命名・共通規約

- 接頭辞: `dim_`（マスタ）/ `raw_`（生データ・入力）/ `mart_`（集計ビュー）。既存BIの規約を踏襲。
- 施設キー: `work_facility`（TEXT・BIコード）。`dim_facility_mapping.facility` と一致。
- 従業員キー: `staff_code`（TEXT・KOTコード）。
- 時間: 分(整数)で保持。表示は時間(h)へ換算。予実・人件費は休憩控除後の実働ベース。
- 由来: `raw_attendance_daily.source` に 'KOT' / 'manual'。

## 2. テーブル定義（要点）

DDLに準拠。実装時の注意点のみ記載。

### 2.1 dim_staff（既存・ALTER）
- 追加列: `wage_type('時給'|'月給')`, `hourly_wage`, `monthly_salary`, `deemed_ot_hours`, `contracted_monthly_hours`, `is_spot`。
- スポット要員は `is_spot=true` で登録し、`hourly_wage` を設定。`kot_code` 相当は持たない（あるいはダミー）。

### 2.2 dim_shift_pattern（新規）
- `pattern_type`: '勤務' or '休日'。
- 勤務: `start_time/end_time/break_minutes/default_role_id` を持ち、予定実働(分)＝(end−start)−break。
- 休日: 時刻はNULL、`is_paid`（有給=true, 公休=false）。
- `color`: 任意色。UIのセル配色に使用。

### 2.3 raw_shift_plan（新規・予）
- 一意: `(staff_code, work_facility, work_date)`（勤怠と同一構造）。
- `pattern_id` でパターン参照（旧status enumは廃止。勤務/休日はパターン種別で判定）。
- `planned_minutes`: パターン既定値を初期投入し、以後ユーザー編集可（コピペ後の時間修正を許容）。
- 分割時: `planned_minutes` = 紐づく `raw_shift_segment.work_minutes` 合計と一致させる（保存時に再計算）。

### 2.4 raw_shift_segment（新規）
- 分割しない日も内部的に1行（seq=1, role=default_role）で持ってよいし、非分割日はセグメント無しで `planned_minutes` 直持ちでもよい（実装選択。推奨は「分割時のみセグメント生成」）。

### 2.5 raw_daily_plan_context（新規）
- 一意: `(facility, work_date)`。`onhand_rooms/forecast_rooms/memo` は手入力。

### 2.6 raw_attendance_daily（既存・再利用）
- 実DBに `source` 列が無いため **T01 で `source text not null default 'KOT'` を ALTER 追加**（既存KOT行は自動的に 'KOT'）。`is_spot` 列は追加しない（スポット判定は `dim_staff.is_spot`）。
- KOT取込に加え、**スポット実働の手入力行**を `source='manual'` で格納（staff は `dim_staff.is_spot=true`）。
- KOT取込のupsertは `(staff_code, work_date, work_facility)`。スポットは staff_code が別系統のため衝突しない。

## 3. 計算ロジック

### 3.1 予実（mart_shift_variance）
```
plan_minutes     = raw_shift_plan.planned_minutes
actual_minutes   = raw_attendance_daily.total_work_min
variance_minutes = actual_minutes − plan_minutes      (NULLは0扱い)
```
FULL OUTER JOIN で「予のみ（欠勤）」「実のみ（計画外出勤）」も検出。

### 3.2 人件費（mart_labor_cost_actual / _plan）
共通の賃金ロジック（実は実働時間、予は計画時間を入力）:
```
時給者:  cost = round(hours × hourly_wage)
月給者:  ot   = max(0, hours − contracted_monthly_hours − deemed_ot_hours)
        cost = monthly_salary + round(ot × (monthly_salary / contracted_monthly_hours) × 1.25)
スポット: cost = round(hours × hourly_wage)   (is_spot=true, 時給扱い)
```
- 割増は 1.25 のみ（深夜・法定休日割増なし）。
- `contracted_monthly_hours` が0/NULLの月給者はゼロ除算回避（NULLIF）。設定必須。

### 3.3 生産性KPI供給
施設×月ロールアップ ビュー **`mart_labor_cost_monthly`**（`mart_labor_cost_actual` を集約）:
```
deemed_ot_excess_pay = sum(ot_pay_over_deemed)   -- みなし残業超残業代
spot_hours           = sum(spot_hours)           -- 派遣・その他労働時間
総労働時間            = sum(raw_attendance_daily.total_work_min)/60  (KOT+manual 両方)
```
**本repoに `mart_productivity` ビューは無い**（生産性は productivity ページでクライアント集計、
手入力2項目は `dim_productivity_manual`）。よって T13 は:
- `mart_labor_cost_monthly` を新設。
- productivity ページの「みなし残業超残業代」「派遣・その他労働時間」を、手入力(`dim_productivity_manual`)から `mart_labor_cost_monthly`(施設×月) 参照へ差し替え。
- 設定画面の当該2項目の**入力UIを廃止**（テーブル列・備考は残す）。
- 総労働時間は既に `mart_labor_monthly`(=raw_attendance_daily 集計) 由来で source を問わず含むため、スポット分も自動反映。二重計上に注意（手入力2項目を必ず除外）。

### 3.4 稼働前提（mart_daily_plan_context）
`budget_daily`（予算）LEFT JOIN `raw_daily_plan_context`（手入力）。
※ `budget_daily` の施設/日付列名を実DBに合わせる。

## 4. 画面仕様（シフト入力）

### 4.1 ツールバー
施設セレクタ / 月セレクタ / 前月コピー（曜日合わせ）/ 短期バイト追加 / パターン・役割設定 / 保存。

### 4.2 稼働前提・メモ（折りたたみ、`<details>`）
4行 × 日付列: 予算(室/人数・自動)、オンハンド(手入力)、予測(手入力)、メモ(手入力・2文字表示＋ホバー全文)。

### 4.3 グリッド本体
- 行: 従業員（フラット・区分バッジ 月給/時給/スポット）。部署グルーピングなし。
- セル: パターン`<select>` ＋ 時間数`<input>`（勤務時のみ活性）。パターン色を反映。
- 右端列: 労働時間計 / 休日数。
- フッタ: 日別 予定合計時間。
- 分割: セルから役割分割エディタを開き `raw_shift_segment` を編集。

### 4.4 月次サマリー
月次 労働時間合計 / 人件費合計 / うち派遣・その他時間。**即時再計算**。

### 4.5 挙動
- パターン選択 → 既定時間・色を投入し集計再計算。
- 時間手修正 → 該当セルのみ更新し集計再計算。
- コピー＆ペースト（Excel風）→ 選択セル群を複製。貼付後の時間修正可。
- 前月コピー → 前月シフトを同一曜日にマッピングして投入。

## 5. データフロー

```
[KOT日別]──取込──▶ raw_attendance_daily(source=KOT) ─┐
[スポット手入力(シフト画面)]─▶ raw_attendance_daily(source=manual,is_spot) ─┤
                                                     ├─▶ mart_labor_cost_actual ─▶ 生産性KPI
[パターン設定]─▶ dim_shift_pattern ─┐                │
[役割設定]─────▶ dim_role ──────────┤                │
[シフト入力]───▶ raw_shift_plan / raw_shift_segment ─┼─▶ mart_shift_variance(予実)
                                                     └─▶ mart_labor_cost_plan(計画人件費)
[日別予算]────▶ budget_daily ─┐
[オンハンド/予測/メモ手入力]─▶ raw_daily_plan_context ─▶ mart_daily_plan_context(稼働前提)
```

## 6. 実装上の注意

- **同一リポジトリ**（既存KPIアプリの拡張）。共有インフラ（Supabaseクライアント/認証/施設・従業員参照/デザイン）を再利用し、シフトは新セクションとして追加。KPI改修（旧手入力の廃止）も同一repoで実施する。
- 時間換算・丸めはビュー側で完結（表示側は再丸めしない）。
- 月給の base（月給固定分）は「月に1回」計上（ビューは staff×月で1行のため自然に1回）。
- スポット導入と同時に、生産性KPI旧手入力の除外を必ず行う（二重計上防止）。
- `budget_daily` の列名確認を初期タスクに含める。
