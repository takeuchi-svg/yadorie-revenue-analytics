# 運用手順書

日常運用（月次の取込・随時のマスタ更新・DDL適用・アカウント管理）の手順。

## 1. 月次ルーティン（毎月やること）

### 1-1. 売上データ取込（各施設）
アップロード画面 `/upload` → 「売上データ」タブ。デフォルト施設を選んでから、その月のCSVをまとめてドロップ。
- 対象: PMS(Staysee) 5種（予約/基本商品/その他商品/入金/販売数集計表）＋ サイトコントローラ(Lincoln) ＋ レート表。
- **ファイル名に対象年月（例 `2026_05`）を含めること**。明細系（基本/その他商品/入金/販売数/レート）は年月が読めないと**取込を拒否**する（再取込での重複倍加を防ぐため）。
- 取込後、結果に「⚠ n行スキップ」が出たら、ヘッダー名の変化やフォーマット差異を確認する（黙って欠落しない設計）。

### 1-2. PL実績取込（各施設）
`/upload` →「PL実績」タブ → 会計の損益計算書CSV。**同一施設×年度を上書き**。

### 1-3. 勤怠取込（全施設一括）
`/upload` →「勤怠」タブ → Touch On Time の日次出力（.xls/HTML）を1ヶ月分（約30ファイル）まとめて。
- 全施設一括のため施設選択は不要。ヘルプ按分・施設マッピングは自動。
- **「⚠ 未マッピング所属 n行スキップ」が出たら新施設/新部署の可能性** → `dim_facility_mapping` と `src/lib/etl/attendance-parser.ts` の `FACILITY_MAP`/`NAME_FAC` に追加が必要。
- 権限: 全施設分を含むため **admin が実施**（member は自施設外の行が拒否される）。

### 1-4. スポット実働（居れば）
シフト画面 `/shift` → スポット行に実働時間を入力 → 保存（実績 `raw_attendance_daily(source='manual')` に記録され生産性KPIへ反映）。

> 生産性の「みなし残業超の残業代」「派遣・その他労働時間」は**手入力不要**（勤怠×賃金から自動算出＝T13）。

## 2. 随時（変化があったときだけ）

- **賃金設定**: 新入社員（勤怠取込で自動的に従業員一覧へ）・昇給・時給改定・スポット追加時に、設定 `/settings`「従業員 賃金設定」で入力。毎月は不要。**給与閲覧権限**が必要（adminは常に可）。
- **稼働日数**: 予実の在庫数算出用。設定「稼働日数（月別）」。
- **OTAマーケ費用**: 設定「OTAマーケティング費用」。
- **施設マスタ（総客室数）・勤務パターン/役割**: 設定（**編集はadminのみ**）。
- **年度予算**: 計画スプレッドシートから取込（budget-import スキル）。

## 3. DDL適用（スキーマ変更）

**直DB接続は現環境で不通のため、Supabase SQL Editor で手動実行する**（`scripts/migrate.mjs` のpg直結は使わない。使う場合は環境変数 `SUPABASE_DB_URL` が必要）。

1. Supabase Dashboard → SQL Editor → New query
2. `scripts/sql/<対象>.sql` の全文を貼り付け → Run（RLS警告は「Run and enable RLS」）
3. 冪等（何度実行しても安全）に作ってある。

主なSQL: `setup.sql`(基盤) / `shift_setup.sql`(シフト) / `shift_seed.sql`(役割・パターン) / `rls_facility.sql`(施設RLS) / `wage_permission.sql`(給与権限・人件費)。適用順は setup → shift_setup → shift_seed → rls_facility → wage_permission。

## 4. アカウント管理

設定「ユーザー管理」（admin のみ）。
- 発行: メール＋初期パスワード＋役割（一般/管理者）。一般は閲覧施設を選択。
- **給与閲覧**: 一般ユーザーの行のチェックで、割当施設の従業員の賃金・個人別人件費の閲覧/編集を許可。
- 一般ユーザーは割当施設のデータのみ（DBのRLSで強制）。給与閲覧OFFなら賃金は非表示・シフトの人件費タイルは「-」。

## 5. デプロイ

main へ push で Vercel が自動デプロイ（takeuchi-svg / beta.vercel.app）。環境変数 `ANTHROPIC_API_KEY`・`SUPABASE_SERVICE_ROLE_KEY` 等は Vercel 側に設定。

## 6. トラブル時

- ページに**赤「データ取得エラー」** = 通信/権限の問題。再読込。→ 黄「データ未登録」（＝未取込）とは別物なので、再アップロードしない。
- 一般ユーザーで何も表示されない = 施設割当が未設定。ユーザー管理で確認。
- DDL実行で権限エラー = 自分がadmin登録済みか `select role from app_user where user_id=auth.uid()` で確認。
