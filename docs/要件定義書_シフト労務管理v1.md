# YADORIE シフト・労務管理システム 要件定義書（v1）

作成: 経営企画本部 / 対象システム: シフト作成 ＋ 予実（対勤怠）＋ 人件費管理
関連成果物: `ddl_shift_labor_v1.sql`（確定DDL） / `仕様書_シフト労務管理v1.md`（システム仕様書） / `shift_ui_mock.html`（UIモック）

---

## 1. 目的

ryokan/hotel 各施設のシフト作成を「速く・考えずに」行いつつ、作成したシフト（予）を既存の勤怠実績（実）と **従業員×日** の粒度で突合し、**労働時間と人件費の予実管理**を成立させる。既存の売上分析・生産性KPI（Supabase）と同一DBに乗せ、二重入力ゼロで KPI にも波及させる。

## 2. 背景

- 現行はExcelの勤務割表。時間帯がパターン化されているが、「忙しい日は1〜2時間残業」といった超過が最初からシフトに組み込まれず、**時間数の予実管理ができていない**。
- 部署（フロント/清掃 等）で人を固定的に分けており、**マルチタスク化の推進**と相性が悪い。
- 一方で、勤怠実績・従業員マスタ・施設マッピング・日別予算は **既にSupabaseに存在**し、生産性KPIが稼働中。シフト側を足すだけで予実が閉じる設計余地がある。

## 3. スコープ

### v1に含む
- シフト作成UI（月グリッド・パターン入力・役割時間帯分割・前月コピー・スポット追加）
- 予実管理（予＝シフト計画、実＝勤怠実績、従業員×日）
- 人件費・残業の自動計算（時給/月給/見込み残業/スポット）
- 稼働前提（予算・オンハンド・予測・メモ）の表示/入力
- マスタ設定（勤務/休日パターン、役割、色）
- 生産性KPI手入力2項目の自動化への切替

### v1に含まない（将来）
- 賃金改定履歴（effective-dated 賃金テーブル）
- 応援(ヘルプ)勤務で施設を跨ぐ場合の予実JOIN厳密化
- 深夜(22時以降)・法定休日の割増計算（1.5等）
- 労基アラート（連勤・インターバル等）、必要人数最適化、通知/出力

## 4. 用語

| 用語 | 定義 |
|---|---|
| 予 | シフト計画上の予定労働（`raw_shift_plan.planned_minutes`） |
| 実 | 勤怠実績の労働合計（`raw_attendance_daily.total_work_min`） |
| 予実 | 予と実の差（実−予） |
| スポット | 派遣・タイミー等の短期要員（KOT打刻なし、実働は手入力） |
| 見込み残業 | 月給に含まれるみなし残業時間。超過分のみ残業代計算 |

## 5. 全体アーキテクチャ

- 位置づけ: **既存KPIダッシュボードと同一リポジトリ・同一Supabase**。利用者から見ればシフト専用の別セクション（別アプリ的UX）だが、コードとDBは共有し、KPI改修も同一repoで行う。
- スタック: 既存踏襲＝**Next.js + Supabase + Vercel**（※旧記述の「GAS/Google Sheets→…」は別プロジェクトからの誤記。本repoは最初からNext.js+Supabase）。
- 施設キー: `work_facility`（TEXT・BIコード） ＝ `dim_facility_mapping.facility` に統一。
- 従業員キー: `staff_code`（TEXT・KOTコード）を背骨とする。
- 実績の所有者は **実績レイヤー（`raw_attendance_daily`）**。シフトアプリは「予の入力」と「KOTが拾えないスポット実働の入力口」を提供するのみ。

## 6. 機能要件

### 6.1 シフト入力
- 月グリッド（従業員行 × 日付列）。**部署でグルーピングしない**（フラット表示、マルチタスク前提。役割はシフト側に持つ）。
- 各セルは **勤務/休日パターンをプルダウンで選択**。選択でパターン既定の開始/終了/休憩と予定実働（分）が自動投入。
- **時間数はセルに表示**し、選択後も **手修正可**（例: 早番8.0h → 9.5h）。超過は視覚的に警告表示。
- **Excel風コピー＆ペースト**でセル複製。貼付後も時間数は個別に修正可。
- **前月コピーは曜日合わせ**（日付番号ではなく同一曜日にマッピング）。
- **役割の時間帯分割**（任意）: 必要な日だけ「7:00–10:00 清掃／10:00–16:00 フロント」のように分割（`raw_shift_segment`）。分割しない日は1コマ=1セグメント扱い。
- **スポット要員の手動追加**（派遣/タイミー）。時給を持たせ人件費自動、実働は手入力で実績レイヤーへ。
- **稼働前提・メモ（折りたたみ可）** を日付行とシフト表の間に配置:
  - ⑴ 予算 稼働室/宿泊人数（`budget_daily.rooms_sold / guests` から表示・自動）
  - ⑵ 現状オンハンド稼働室（手入力）
  - ⑶ 予測稼働室（手入力）
  - ⑷ メモ（手入力・イベント等）。表示は先頭2文字程度、ホバーで全文ポップアップ。

### 6.2 予実管理
- 突合単位 = `従業員 × 日付 × 施設`。予＝`planned_minutes`、実＝`total_work_min`（いずれも休憩控除後の実働・分）。
- ビュー `mart_shift_variance` が差分（`variance_minutes = 実 − 予`）を算出。
- 実績が入った後、各セルに予実差を重畳表示できる。

### 6.3 人件費・残業計算
- 時給者: `実働時間 × 時給`。
- 月給者: `月給（固定）＋ 残業代`。
  - 残業代 = `max(0, 月労働時間 − 月所定 − 見込み残業) × 残業単価`
  - 残業単価 = `月給 ÷ 月所定 × 1.25`（**深夜・法定休日割増は含めない**）。
- スポット: `実働時間 × 時給`。
- 予（計画）と実（実績）それぞれで算出（`mart_labor_cost_plan` / `mart_labor_cost_actual`）。人件費も予実で見える。
- **人ごとの人件費はUIに表示しない**（内部計算のみ）。表示は月次合計。

### 6.4 生産性KPI連携（手入力の自動化）
> 【本repo実態】`mart_productivity` ビューは存在せず、生産性KPIは productivity ページで
> クライアント集計。手入力2項目は `dim_productivity_manual`（設定画面）。よって連携は
> 「集計ビュー `mart_labor_cost_monthly` を新設 → ページの参照元を差し替え → 手入力2項目UIを廃止」で行う。

- 施設×月ロールアップ **`mart_labor_cost_monthly`** を新設（`mart_labor_cost_actual` を集約）:
  - みなし残業超の残業代 = `sum(ot_pay_over_deemed)` → 列 `deemed_ot_excess_pay`
  - 派遣・その他労働時間 = `sum(spot_hours)` → 列 `spot_hours`
- productivity ページの当該2指標を、`dim_productivity_manual` 手入力から **`mart_labor_cost_monthly` 参照**へ差し替える。
- `dim_productivity_manual` の当該2項目の**入力UI（設定画面）は廃止**。テーブル列や備考(dispatch_other_notes 等)は残す（過去値・メモ保全）。
- スポット実働は `raw_attendance_daily`（`source='manual'`）に格納。**スポット判定は `dim_staff.is_spot`**（attendance に is_spot 列は持たない）。既存KOT行は `source='KOT'`。staff_code が別系統のため衝突しない。
- 切替時の必須作業: KPI計算式から**旧・手入力2項目を除外**（二重計上防止）。T11(スポット)の後にT13で実施。

### 6.5 マスタ設定
- 勤務パターン・休日パターン（`dim_shift_pattern`）を**設定画面で追加**。開始/終了/休憩/既定役割/**色**を指定。休日パターンのデフォルトは公休・有給。
- 役割（`dim_role`）を**設定画面で追加**（色指定可）。

### 6.6 集計・表示
- 個人別: 労働時間計、休日数。
- 日別: 予定合計時間。
- 月次: 労働時間合計、人件費合計、うち派遣・その他時間。
- **数値は即時反映**（セル編集で関連集計がリアルタイムに再計算）。

## 7. 非機能要件

- 性能: 月グリッド編集時の集計は即時（クライアント側再計算＋確定時にDB保存）。
- 整合: 予実・人件費・KPIは同一の実績テーブル/ビューを参照し単一ソースを保つ。
- 権限: 施設ごとの編集権限（総支配人＝自館、本部＝全館）。**RLSで実装済み**（`is_admin()`/`can_access_facility()`＝rls_facility.sql、賃金は `can_view_wage()`＝wage_permission.sql）。詳細は [`データベース全体設計.md`](データベース全体設計.md) §権限。
- 監査: `updated_at`、`source`（KOT/manual）で由来を保持。
- 拡張性: パターン・役割・色はデータ駆動（コード変更不要で追加）。賃金履歴・応援跨ぎ・割増はv2で拡張可能な構造。

## 9. 主要計算仕様（式）

```
予実差(分)         = 実(total_work_min) − 予(planned_minutes)          [従業員×日×施設]
時給者 人件費      = 実働時間 × 時給
月給者 人件費      = 月給 + max(0, 月労働時間 − 月所定 − 見込み残業) × (月給 / 月所定 × 1.25)
みなし残業超残業代 = 上式の残業代部分（月給者のみ）
派遣その他労働時間 = is_spot=true の実働時間合計
```
