# ドキュメント地図（最初に読む）

YADORIE Revenue Analytics のドキュメント体系と「どれが正本か」を定義する。**迷ったらまずこれ。**

## SSOT（Single Source of Truth）の原則

| 対象 | 正本（ここが正しい） | 補足文書（背景・意図） |
|---|---|---|
| **DB全体（実在するテーブル/ビュー/キー/前提）** | [`データベース全体設計.md`](データベース全体設計.md) ＋ 実DB | 各機能の設計書 |
| DDL（実行するSQL） | `scripts/sql/*.sql`（冪等・SQL Editorで実行） | `docs/ddl_shift_labor_v1.sql`（シフトの設計時案。実行は scripts 版） |
| 売上BIのraw層スキーマ | `scripts/migrate.mjs` の DDL 文字列 | `データベース設計書.md`（※一部古い。全体設計.md が優先） |
| 運用（月次取込・DDL適用・権限） | [`運用手順書.md`](運用手順書.md) | — |
| AIエージェントが読む前提 | `CLAUDE.md` ＋ 全体設計.md | 各要件定義書 |
| UI/ページ構成 | 実装（`src/app/(dashboard)/*`）＋ 本地図の下表 | `UI仕様書.md`（※ページ数が古い） |

## ページ一覧（現行・実装が正）

サイドバー3+管理。ルートは `src/app/(dashboard)/<name>/page.tsx`。

- **ビュー**: 概要`/` / 売上分析`/revenue` / 日別売上`/daily`
- **分析**: レートコントロール`/rate` / CXL＆LT`/cancel` / 料飲`/fb` / OTA`/ota` / 予実管理(PL)`/yojitsu` / 生産性`/productivity` / シフト・労務`/shift`
- **管理**（左下固定）: アップロード`/upload` / 設定`/settings`

## as-built 更新ルール（重要）

**実装が設計と乖離したら、CLAUDE.md にメモを足すだけで終わらせず、該当docと本地図・全体設計.md に反映する。**
- 「本repo実態と設計の差分」は CLAUDE.md の一時置き場ではなく、正本（全体設計.md）へ還流する。
- 古くなった記述は消すか、冒頭に「⚠️ 更新: 正は○○を参照」の注記を付ける（本棚卸しで各docに付与済み）。
- 新テーブル/ビュー追加時は必ず 全体設計.md の台帳に1行足す。
